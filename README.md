# Selected Projects

## **[Habitat Picker](https://github.com/Eng-ADAL/Habitat_Picker)**

Collaborative cloud-based ETL project built around the **UK Land Registry Price Paid Dataset**.

The pipeline ingests raw public data, cleans and normalises it through a structured Python workflow, and writes analytics-ready datasets to Amazon S3 using partitioned Parquet files.

Highlights:

- ingestion and transformation pipeline written in Python
- raw and curated data zones stored in Amazon S3
- partitioned Parquet datasets for efficient analytics
- schema enforcement and data quality checks
- designed for integration with BI tools and dashboards

Tech:

`Python` `Amazon S3` `Parquet` `SQL` `Git`

This project was built collaboratively.  
Analytics, visualisation, and collaboration details are available in the repository.

---

## **[Obscured-by-Clouds](https://github.com/Eng-ADAL/OBC)**

A modular **ETL pipeline framework** designed to transform raw transactional data into structured datasets ready for relational storage.

The goal is production-style data platform with clear pipeline stages and reproducible execution.

Features:

- Extract–Transform–Load architecture
- schema validation and logging
- Dockerised runtime
- modular pipeline components
- relational data modelling workflow
- one-click setup across Windows, macOS, and Linux environments

Tech:

`Python` `PostgreSQL` `Docker` `SQL`

---

## **[MindAtlas](https://github.com/Eng-ADAL/MindAtlas)**

A data pipeline that extracts personal knowledge data from the **Joplin REST API**, transforms it into structured datasets, and loads it into a local analytical database.

Focus areas:

- API ingestion and pagination handling
- structured transformation workflows
- lightweight analytical storage
- reproducible processing scripts
- Connecting MCP server for auto-documentation

Tech:

`Python` `REST API` `Pandas` `DuckDB`

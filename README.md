## Selected projects

### **[Habitat Picker](https://github.com/Eng-ADAL/Habitat_Picker)**
Collaborative ETL project built around the UK Land Registry Price Paid Dataset.

It ingests raw public data, cleans and normalises it through a Python workflow, and writes analytics-ready datasets to Amazon S3 as partitioned Parquet files.

**Highlights**
- ingestion and transformation pipeline written in Python
- raw and curated data zones stored in Amazon S3
- partitioned Parquet datasets for efficient analytics
- schema enforcement and data quality checks
- designed for integration with BI tools and dashboards

**Tech:** Python, Amazon S3, Parquet, SQL, Git

### **[Obscured-by-Clouds](https://github.com/Eng-ADAL/OBC)**
A modular ETL pipeline framework designed to transform raw transactional data into structured datasets ready for relational storage.

The goal is a production-style data platform with clear pipeline stages and reproducible execution.

**Features**
- Extract, Transform, Load architecture
- schema validation and logging
- Dockerised runtime
- modular pipeline components
- relational data modelling workflow
- one-click setup across Windows, macOS, and Linux

**Tech:** Python, PostgreSQL, Docker, SQL

### **[MindAtlas](https://github.com/Eng-ADAL/MindAtlas)**
A data pipeline that extracts personal knowledge data from the Joplin REST API, transforms it into structured datasets, and loads it into a local analytical database.

**Focus areas**
- API ingestion and pagination handling
- structured transformation workflows
- lightweight analytical storage
- reproducible processing scripts
- MCP server integration for auto-documentation

**Tech:** Python, REST API, Pandas, DuckDB

### **[Status Checker](https://github.com/Eng-ADAL/check_status)**
A containerised service that monitors a website and records status and health data in a database.

Built as a simple monitoring service with containerised execution.

**Features**
- Dockerised Python runtime
- monitoring loop
- database persistence layer
- easy to extend for alerts or metrics

**Tech:** Python, Docker, PostgreSQL

### **[experimental_linux](https://github.com/Eng-ADAL/experimental_linux)**
A modular Debian workstation bootstrap for reproducible, explicit, and maintainable Linux setup.

It turns a fresh Debian system into a usable workstation with base CLI tools, dotfiles, optional desktop environments, and custom utilities.

**Focus areas**
- reproducible Linux setup
- modular installation flow
- dotfile management
- desktop and CLI tooling
- workstation automation

**Tech:** Bash, Debian, Linux, tmux, Vim, Zsh

### **[dumpdir](https://github.com/Eng-ADAL/dumpdir)**
A small command-line tool for exporting a directory tree and selected file contents in a controlled way.

Built for sharing project context without exposing the whole filesystem.

**Focus areas**
- deterministic output
- safe filesystem snapshots
- automation-friendly CLI design
- secret-aware exporting
- human-readable and machine-readable output

**Tech:** Bash, Linux, CLI tooling
